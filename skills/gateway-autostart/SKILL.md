
# Gateway 自动重启 & 开机自启动配置技能

## 功能说明

一键配置 OpenClaw Gateway 的两项核心保障机制：
1. **自动重启**：网关进程意外停止或执行 `restart` 命令后，Windows 计划任务自动拉起
2. **开机自启动**：电脑开机后自动启动 Gateway，无需手动操作

## 适用场景

- ✅ 新装 OpenClaw 环境
- ✅ 重装系统后恢复配置
- ✅ 修复网关重启后不自动恢复的问题
- ✅ 长期无人值守的服务器/主机

---

## 配置步骤（必须手动操作，需要管理员权限）

### 第一步：打开任务计划程序

按 `Win + R`，输入 `taskschd.msc`，回车打开**任务计划程序**

---

### 第二步：配置自动重启

1. 在**任务计划程序库**中找到 `OpenClaw Gateway` 任务
2. 右键 → **属性**
3. 切换到 **设置** 标签页
4. 勾选 ✅ **如果任务失败，按以下频率重新启动**
   - **重启间隔**：选择 `1 分钟`
   - **重启尝试次数**：填写 `3`
5. ⚠️ 可选（笔记本电脑建议）：取消勾选 ❌ **如果计算机改用电池电源，则停止**
6. 点击 **确定** 保存

---

### 第三步：验证开机自启动（默认已开启）

1. 切换到 **触发器** 标签页
2. 确认有一个 **启动时** 的触发器
3. 如果没有，点击 **新建**：
   - 开始任务：选择 **启动时**
   - 勾选 ✅ **启用**
   - 点击 **确定**

---

### 第四步：验证配置

打开 PowerShell，执行以下命令验证配置：

```powershell
# 查看计划任务配置
$task = Get-ScheduledTask -TaskName "OpenClaw Gateway"
$task.Settings | Select-Object RestartCount, RestartInterval, StopIfGoingOnBatteries
```

**预期输出：**
```
RestartCount RestartInterval StopIfGoingOnBatteries
------------ --------------- ----------------------
           3 PT1M                              False
```

---

### 第五步：测试自动重启

```powershell
# 执行网关重启，观察是否能自动恢复
openclaw gateway restart
```

- 网关会先关闭
- 等待约 1 分钟
- Control UI 会自动恢复连接
- 如果看到"System: Your previous turn was interrupted..."提示，说明自动重启成功

---

## 配置完成后的效果

| 场景 | 之前 | 之后 |
|------|------|------|
| 执行 `gateway restart` | ❌ 网关关闭后死掉，需手动启动 | ✅ 1分钟后自动恢复 |
| 网关进程崩溃 | ❌ 无法自动恢复 | ✅ 自动重试3次 |
| 电脑重启后 | ❌ 需手动启动网关 | ✅ 开机自动启动 |
| 笔记本拔电源（可选） | ❌ 网关停止 | ✅ 继续运行 |

## 故障排除

### ❌ 执行 restart 后超过 2 分钟仍未恢复

1. 打开任务计划程序，检查 `OpenClaw Gateway` 任务状态
2. 如果状态是"正在运行"，说明网关起来了但连接有问题，刷新网页即可
3. 如果状态是"准备就绪"，手动右键→**运行**启动

### ❌ 开机后网关没自动启动

1. 检查触发器是否设置了 **启动时**
2. 检查 **条件** 标签页，确认没有勾选"仅当计算机使用交流电源时才启动任务"
3. 检查 Windows 安全软件是否阻止了计划任务

### ❌ RestartInterval 显示为空

确认在设置里确实勾选了"如果任务失败，按以下频率重新启动"，并填写了数值。有些 Windows 版本需要先点击"应用"再点"确定"才能生效。
